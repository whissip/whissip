{{{Upgrading from Manywhere Miniblog}}}

{{{Before you begin, please note that some network features of b2evolution require you to agree our simple Terms of service.}}}

{{{As always, it is wise to do a backup of your current website / blog / database (depending on what you already have) before starting to install something new.}}}

{{{Once you have unzipped the b2evolution distribution:}}}

  1. {{{On your server, locate the mysql database you are already using for Manywhere Miniblog. You must use the same database for b2evolution to be able to upgrade. Feel safe, b2evolution will not alter your Miniblog tables and data in any way!}}}

    {{{Note 1:}}} {{{Unless you changed the config, all tables created by b2evolution are prefixed with evo_ . Unless you already have an application using the same prefix, this makes it safe to install b2evolution in a database shared with other applications. For example, Miniblog uses two tables named miniblog and user. Your existing tables will not be affected in any way.}}}
    {{{Note 2:}}} {{{Why the same database? Because the SQL process we use to copy the data -- which is fast and secure -- can only be done within a single database.}}}
  2. {{{Upload the b2evolution files to your website.}}}

    {{{Note:}}} {{{Only the contents of the /blogs directory need to be uploaded to your site. If you want b2evolution to handle the homepage of your site, upload the contents of the /blogs folder to your website root. However, you could start by uploading the files to a new subdirectory, /b2evo for example; that way your existing Miniblog remains fully functional while you check out b2evolution.}}}
    {{{Let's assume you install the contents of the blogs folder to http://yoursite.com/b2evo ...}}}
  3. {{{Run the installation script on your server. It is located in http://yoursite.com/b2evo/install . Then, simply follow the on screen instructions.

    The installer will ask you a few questions, create required database tables, copy your Miniblog data to b2evo (without affecting Miniblog) and configure b2evolution for you.}}}
  4. {{{Check to make sure the new installation works.}}}
  5. {{{On production servers, it is recommended that you delete the /install subfolder.}}}
  6. {{{After basic install is done, there are a few tricky things you may need to take care of (online manual).}}}
