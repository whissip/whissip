Upgrading from b2evolution version 0.8.6 or later (prior to 0.8.9)

Upgrading from version 0.8.6 is easy:

  1. You can keep your skins, blog templates and stubfiles for your blogs. However, now may be a good time to back them up ;) You should also backup your database if you can.
  2. Backup your /conf folder, copy the new conf files and redo your configuration in the /conf folder as you did previously. As always, the only requirement is to edit _config.php. The other config files can be used without modification (provided you use the new ones!), however you should have look inside for fine tuning once your new version is running. Please note: there is no _antispam.php config file no longer. Antispam processing has been moved to the database.
  3. Replace the b2evolution core files. That means the following folders: /admin, /b2evocore, /htsrv, /locales, /xmlsrv. You also need to temporarily copy the /install folder to your site.
  4. Upload the new files to your server, and then run the database tables upgrade on your server!
  5. On production servers, it is recommended that you delete the /blogs/install/ subfolder.
  6. Just in case you did not do it before, you may want to check out the tricky stuff. (online manual)
