Upgrading from the original b2

The upgrade procedure below is for upgrading an original b2 installation. If you are upgrading from an older version of b2evolution, this does not apply to you.

The upgrade procedure below has been tested exclusively with b2 versions 0.6.1 and 0.6.2. Though, I think versions 0.5.x should also upgrade without problems. Anyway, if you run any old version of b2, you can give it a try quite safely. b2evolution will not harm your data, even if it can't understand it. The worst you would get is an error message.

As always, it is very wise to do a backup of your current website / blog / database (depending on what you already have) before starting to install something new.

Once you have unzipped the b2evolution distribution:

  1. On your server, locate the mysql database you are already using for b2. You must use the same database for b2evolution to be able to upgrade. Feel safe, b2evolution will not alter your b2 data in any way!

    Note 1: Unless you changed the config, b2evolution uses tables prefixed with evo_ where the orginial b2 used tables prefixed with b2.
    Note 2: Your existing b2 tables and data will not be altered in any way by b2evolution. b2evolution only copies your existing data from b2 tables to new evo_ tables.
    Note 3: Why the same database? Because the SQL process we use to copy the data -- which is fast and secure -- can only be done within a single database.
  2. On your computer, edit the file /blogs/conf/b2evo_config.php with a text editor and set the parameters according to your current configuration. There are basically 3 things to set:
    • Your e-mail address
    • Access information to your database (hostname, basename, login, password)
    • The URL where you are going to run b2evolution

      Note: It is recommended that you do a first test installation and/or upgrade by uploading the whole demo structure to your website, starting with the folder b2evolution. This way, your new blogs will have an URL of the form: http://www.yoursite.com/b2evolution/blogs/index.php, http://www.yoursite.com/b2evolution/blogs/blog_b.php, etc.
      This way, you also leave your existing b2 running safe as is, until you feel comfortable with the upgrade process to work as you expect.
      Of course you can change this later. By then, you will probably want to upload just the contents of your "blogs" folder, to the exact place your original b2 blog was running.
  3. Upload the files to your website, according to the URL you have just configured.
  4. Come back to this page on your server, (should be something like http://www.yoursite.com/b2evolution/doc/upgradefrom_b2.html) and run the database tables installation. This will allow you to create database tables for b2evo and to copy of your existing b2 data into them.
  5. You can now read the blogs or log into the back-office and start playing!
  6. On production servers, it is strongly recommended that you delete the /blogs/install/ subfolder.

  7. After basic install is done, there are a few tricky things you may need to take care of. (online manual)

  8. To upgrade your existing b2 template, please read the online manual about upgrading your b2 template. (online manual)
